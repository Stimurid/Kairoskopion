## 1. Why P7.2C Was Needed

P7.2B source authority recovery used `git add -f` to commit seed outputs under
`data/seed_registry/`. This worked for the immediate commit but left a structural
defect: any NEW seed outputs generated by SeedRegistryWorkflow would be invisible
to git and require manual force-adding. This breaks the reviewable-output contract.

## 2. P7.2B Artifact Audit

All 19 P7.2B files are correctly tracked. The force-add workaround succeeded for
the specific commit but didn't fix the root cause.

## 3. `.gitignore` Problem Confirmed

Line 36: `data/` ignores the directory itself. Git never traverses an ignored
directory, so `!data/seed_registry/` negation at line 65 never takes effect
for new files. Verified with `git check-ignore` on a hypothetical path.

## 4. Structural Fix

Changed `data/` → `data/*`. This ignores the *contents* but not the directory,
allowing git to traverse `data/` and evaluate per-entry negation rules.

Consolidated the data section with clear curated/ignored grouping.
